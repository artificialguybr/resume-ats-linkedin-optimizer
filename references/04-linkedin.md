# LinkedIn Profile Optimization

LinkedIn is a **talent search engine**, not an ATS. A recruiter runs a query and
every profile is scored against it — so optimize for recruiter search ranking
*and* human, in-feed reading. Port the resume's keyword spine here, then adapt
for LinkedIn's field weighting and conversational tone.

> Note on numbers: LinkedIn doesn't publish exact algorithm weights. Multipliers
> below (5x, 40x, etc.) are well-corroborated coach/agency figures echoing
> LinkedIn's own older talent-blog stats — directional, not official constants.
> Character limits and the completeness mechanics are consistently reported.

## How ranking works

- **Text-match first, then relevance weighting.** The algorithm can't infer
  experience — if a keyword isn't literally on your profile, you're invisible for
  it. Profiles are then ranked by keyword placement/density, completeness,
  endorsements, and recency.
- **Two separate recruiter search fields:**
  - **Title field** — searches only current/past **job titles**. Your target
    role must be an actual title/headline, not just buried in prose.
  - **Keywords field** — searches the whole profile.
- **Headline is weighted heaviest** (~5x other fields). Pinned top-3 skills carry
  heavy weight too.
- **Completeness ("All-Star") boost** — All-Star profiles surface far more often;
  updating the current position is one of the biggest visibility levers.
- **Recency/activity** — "recently active" (past 30 days) ranks higher; posting
  ~1×/week and refreshing the profile helps.
- **Endorsements as a tiebreaker** — a skill with 50+ endorsements outranks the
  same skill with 5.
- **Recruiter "Spotlights"** boost response likelihood: Open to Work, "engaged
  with talent brand," recently active, past applicants — plus an AI
  interest-likelihood score layered on top.

### Headline (highest-leverage field)
Never leave the default "Job Title at Company." Front-load role + value in the
first ~70 chars; use `|` or `•` separators; repeat the target job title verbatim.
Formulas:
1. **Role + Specialty + Value** — `Senior Product Manager | B2B SaaS & Fintech | Shipped products used by 2M+ users`
2. **Role + Outcome + Proof** (best for inbound recruiter outreach) — `SDR | 220 meetings booked in 2026 | 4x quota`
3. **Keyword Stack + Value** — `Data Scientist | Python, ML, LLMs, MLOps | Turning models into revenue`

### About / Summary
First-person, conversational. Structure:
1. **Hook (first 2–3 lines / ~300 chars)** — must earn the "see more" click; lead
   with a bold value statement or "who I help + how."
2. **Body (3–5 short paragraphs)** — story + quantified wins; weave target-role
   keywords naturally (5+ role keywords → more searches).
3. **CTA** — how to reach you and what you want next.

Use line breaks (57%+ of views are mobile). Recommended length ~1,800–2,200 chars.

### Experience (vs resume bullets)
Public, algorithm-indexed + human-read, no length pressure. Add context/story
("the why"). **Attach media** (decks, case studies, portfolio, videos) — a native
advantage resumes lack. Per role: a 2–3 sentence scope summary + 3–6 CAR bullets
(current role up to ~8, older 2–3), each action-verb + quantified result. Reframe
resume bullets into slightly more human phrasing; don't copy-paste verbatim.
Repeat the target title in the role title and bullets.

### Skills
Cap 100; be selective at **20–30 targeted** skills. Mix ~40–50% technical,
20–30% functional, some industry + a few soft. **Pinned top 3** carry the most
weight — order them: (1) target role/function, (2) strongest technical
differentiator, (3) high-demand industry skill. Concentrate endorsements on your
top ~10; pursue skill assessments/verified badges for the pinned three (failures
are private). Mine 15–20 target postings and mirror the most-frequent skills.

### Featured / Recommendations / URL / Banner / Photo
- **Featured:** 3–5 items — best posts, portfolio links, case studies. A curated
  proof shelf.
- **Recommendations:** 5–10 quality ones from managers, peers, reports, clients.
- **Custom URL:** `linkedin.com/in/firstname-lastname`; keep < ~20 chars; no
  numbers/birth years; changeable 5×/6 months.
- **Banner:** 1584×396 (4:1); keep text in the central ~1260×300 safe zone.
- **Photo:** 400×400 min (800×800 better), face 60–70% of frame, shoulders-up,
  front-facing; drives large view/message lifts.

## Keyword strategy: LinkedIn vs ATS (the differences)

- **Same keyword spine, different surfaces.** Fix the resume for ATS first, then
  port the exact terms into LinkedIn.
- **Field placement matters more on LinkedIn** — target title must be an actual
  title/headline (Title field ≠ Keywords field). On an ATS, mere presence matters
  more than which section.
- **Repetition + placement compound** on LinkedIn (headline ~5x + About + top-3
  skills + a job title). An ATS mostly checks presence/frequency.
- **Endorsements, activity, completeness are ranking inputs** on LinkedIn — none
  exist in an ATS.
- **Human co-reading:** avoid the raw keyword dumps that might scrape past an ATS
  — profiles are read in-feed by people.

## Open to Work nuance

Public green banner boosts responses (~35–40% more) and recruiters read it as a
neutral availability signal — but some hiring managers read risk into it.
**"Recruiters only"** surfaces you in Recruiter search without the public frame.

## Resume ↔ LinkedIn differences (quick table)

| | Resume | LinkedIn |
|---|---|---|
| Audience | Private, ATS + one recruiter | Public, search engine + many |
| Voice | Terse, no pronouns | First-person, conversational |
| Length | 1–2 pages | No pressure; longer OK |
| Keywords | Exact JD match | Industry-wide + target role, repeated strategically |
| Media | None | Decks/links/video encouraged |
| Currency | Per-application | Always current |

## Common mistakes
Default/weak headline · headline aimed at the past · no About hook (lost at the
~300-char fold) · missing keywords (invisible) · under-All-Star (no photo, <5
skills, no current-role description) · copy-pasting resume bullets · top-3 skills
not aligned to target · inactivity · desktop-only formatting · internal jargon
instead of searchable terms · placeholder/number-laden custom URL.

## All-Star checklist
- [ ] Professional photo + banner (safe zone respected)
- [ ] Keyword-rich headline (220 chars used, front-loaded)
- [ ] About with hook + story + keywords + CTA
- [ ] Current role filled with description + media
- [ ] ≥5 (ideally all relevant) past positions with CAR bullets
- [ ] 20–30 skills; top 3 pinned to target role; endorsements concentrated
- [ ] Education, certifications, featured, 5–10 recommendations
- [ ] Custom URL; location & industry set
- [ ] Open to Work configured (public or recruiters-only per preference)
